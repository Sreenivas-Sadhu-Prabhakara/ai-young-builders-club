### Slide 3 — From clicking to CODING

> **Week 2:** you *clicked* a chatbot. 🖱️
> **Week 3:** you **command** one with code. ⌨️
> More power. More control. You're the builder now.

**Say:** "Clicking is using. Coding is *commanding*. Today you cross over from user to
builder."

---

### Slide 4 — What's an API? (the restaurant)

> ## An API is how one program asks another to do something.
> 🧑‍💻 **You / your code** = order what you want
> 🧾 **The waiter + menu = the API** = the way to ask
> 🍳 **The kitchen = the AI model** = does the real work

**Say:** "You never walk into the kitchen. You ask through the waiter. Our code asks the AI
**model** through its **API** — and it sends an answer back."

---

### Slide 5 — The API key is a secret 🔑

> ## API key = a secret password for code.
> Like a members-only card. **The facilitator holds it. You never touch it.**

**Say:** "A key lets code use the AI. It's a secret — never share it, never paste it. In our
club, *I* hold the key. Same golden rule as always: **no secrets into an AI.**"

---

### Slide 6 — The words we'll use

> - **Python** = a friendly language that reads almost like English
> - **Colab** = run Python in your browser, nothing to install
> - **Notebook** = a doc of runnable blocks called **cells** ▶️
> - **Variable** = a labeled box that holds a value, like `name = "Robo"`

**Say:** "Four words. You'll use all of them in a few minutes — and you'll change some
variables to command your own AI."

---

### Slide 7 — Your mission (only Cell 2!)

> ## Bring YOUR assistant to life
> 1. **Run Cell 1** — Setup (don't touch it)
> 2. **Edit Cell 2** — your `ASSISTANT_NAME`, `PERSONALITY`, `PURPOSE`, `RULE` ✏️
> 3. **Run Cell 3** — the brain loop
> 4. **Run Cell 4** — CHAT with your bot! 💬

**Say:** "You only edit **Cell 2** — your personality. Same code as everyone else; change
Cell 2 and it becomes *yours*. Ready? Build!"
