# Overview & pitch

*The 5-minute read for anyone deciding whether to run or support this club.*

## What it is

**AI Young Builders Club** is a four-session, hands-on AI literacy program for kids aged
**11–13**. Each session is **one hour**, run on consecutive weekends. Kids work on laptops
with real (supervised) AI tools, and across the four weeks each one designs, builds, and
presents **their own AI assistant**.

## The problem it solves

Today's kids use AI constantly but understand it rarely. The result is two bad extremes:

- **Over-trust** — believing whatever a chatbot says, pasting in personal information,
  treating generated images as real.
- **Mystery & exclusion** — feeling AI is "not for me," something only experts touch.

This club replaces both with **literacy and agency**: a working mental model, deliberate
and safe usage habits, and the confidence that comes from having *built* something.

## What makes it work

- **Make first, explain second.** Kids train a model before we define "training," prompt a
  bot before we explain "tokens." The concept lands on top of a real experience.
- **One capstone, four layers.** Motivation stays high because every hour visibly advances
  *their* project. (Purpose → Personality → Code → Showcase.)
- **Timeboxed to the minute.** Each session has a facilitator run-sheet so an hour is never
  wasted. Short concept, long hands-on.
- **Safe by design.** Supervised tools, no personal data in prompts, facilitator-managed
  accounts, and a full session on responsible use.
- **Inclusive but stretchy.** No prior coding needed; keen kids get optional "Go Further"
  challenges.

## Who runs it

One facilitator can run a group of up to eight. You do **not** need to be an AI expert —
the facilitator run-sheets give you minute-by-minute talking points, the answers to common
kid questions, and a fallback plan for when the tech misbehaves. If you can follow a
recipe and stay curious alongside the kids, you can run this.

## What kids walk away with

By the end of Week 4, every kid can:

- Explain what AI is — that it learns from examples and can be confidently wrong.
- Write a clear prompt and judge whether an answer is trustworthy.
- Show their own AI assistant, running from a few lines of Python they can read.
- Name real AI risks (bias, deepfakes, privacy, hallucination) and one way to stay safe.

## What it costs to run

- **Time:** 4 hours of sessions + ~1 hour prep per session for a first-time facilitator.
- **Tech:** One laptop per kid with internet; a projector/screen for the facilitator.
- **Tools:** Mostly free/browser-based (Teachable Machine, Google Colab). Week 2–3 need a
  facilitator-managed account or key for the chat/code tools — see
  [Setup & accounts](for-facilitators/setup.md).

## The arc at a glance

| Week | Theme | Hands-on "wow" | Capstone layer |
|------|-------|----------------|----------------|
| 1 | What even *is* AI? | Train a model in the browser | Pick a **purpose** |
| 2 | Talking machines | Prompt a bot, generate images | Design a **personality** |
| 3 | A taste of real code | Python that calls an AI | Bring it to **code** |
| 4 | Smart & safe + showcase | Red-team your bot | **Showcase** |
